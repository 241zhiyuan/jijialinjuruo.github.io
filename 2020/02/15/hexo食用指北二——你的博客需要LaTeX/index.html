此时，你的hexo博客是能够访问的。
然而，你似乎在想：这玩意真难搞，文章哪里可以上传呢，真麻烦。。。

嗯…我不会告诉你我以前也不会上传文章的还真有这么个问题。
所以，我们今天先说说怎么上传文章。

1.写/上传文章

在此之前，先推荐一个很好的插件：
使用相对路径访问图片插件——hexo-asset-image
jjl推荐与typora一起使用，待会你就知道为什么了。
打开/你的博客文件夹/名字/source/，有没有发现里面有一个文件夹叫_posts？打开它，你又会发现里面有个文件叫hello-world.md是吗？然后你又会发现博客上也有一篇东西叫hello-world。
由此可得：这是存储文章的文件夹。

声明：只有_posts是博客自带的，其他的文件夹都是jjl自己做的，这个将在hexo食用指北三——为你的博客加入额外功能中详细介绍。

1.1 相对路径访问图片

运行代码npm install hexo-asset-image --save
可能有人会问：相对路径有何用处？
那样子你就不需要上传到某些无良图床上，也不用付费了！岂不美哉？
汝听，人言否？
这样能方便的调用图片等资源。
但这还不够。
打开站点配置文件(已在第一篇介绍过——_config.yml)

注：此处应用sublime打开文件，因为typora仅仅只能用于写文章。

按下Ctrl+F(如果不是查找的快捷键请另用快捷键)，搜索post_asset_folder:，将false改成true。
切记，冒号后面加空格！否则会编译失败!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!

在前面，jjl说typora应与这个插件一起食用，为什么呢？
这里jjl来放几张图来告诉大家。

不需要手打路径了！
是不是很方便呢？
如果你不会markdown，可以在站内搜索jjl的《markdown入门》或度娘。

1.2 上传文章

写好一篇文章后，一定要另存为(哦对了，写文章时一定要养成一个好习惯——时不时地Ctrl+s)到_posts。
之前介绍过了hexo g和hexo d，然而这还不够，还需要在两条命令前加入这么一个命令hexo clean以清理缓存，否则会很卡的！
(注：可以用&&来连接每个命令)
等一段时间，博客上就会显示你的文章了！
