hexo食用指北一——安装与基本配置

发表于 2020-02-06 更新于 2020-02-12 分类于 Hexo install,use and deploy 阅读次数：
本文字数： 1.9k 阅读时长 ≈ 2 分钟

有一天，jjl在网上搜东西。
忽然，看到了hexo。
于是他在网上查了hexo是什么东西。
原来是一个博客框架啊！

先放上官网链接

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。 —摘自官方

然而凡事总会有缺点：上传麻烦，域名太长(可以缩短如.xyz,但是只有土豪才会改)……

不过，hexo对于我这种蒟蒻不失于一个不错的选择。

0.准备工作

首先，确保你安装了以下几个软件或拥有以下网站的账号:
Git
Node.js
Github
一个编辑器(建议使用Typora,Sublime)

1.开始搭建博客

1.1 配置SSH

首先搜索git-bash，输入以下代码：


按4次回车，就可以关程序了(会跳出一个随机图，不用管它)
接着打开文件管理器(此电脑)，搜索.ssh，然后打开文件夹，点开里面的pub文件。

然后来到GitHub官网，点击右上角并打开Settings。

点击New SSH Key，随便写一个Title，并把SSH粘贴进Key去。

于是，SSH就配置完成了。

打开hexo官网，中间有一行代码，就是安装代码了。
先cd blog一下。
为了方便，我把代码放出来：



安装好后可能会有一些warn，都不用管它。
对了，还有一个事情。
$\color{red}{\Huge{\text{千万不要关cmd！千万不要关cmd！千万不要关cmd！}}}$
重要的事情说三遍。

拖动hexo官网到最底端，有一行代码hexo init blog，这就是创建博客的核心代码了，因为它就是安装博客的代码了。
可以把代码里的blog改成你喜欢的名字，也尽量用英文。我改成了jjl


运行代码，如果出现了$\color{green}{INFO}$ $Start blogging with Hexo!$，说明安装成功。
这时，cd到你的博客文件名，对我来说就是cd jjl，以后对博客进行操作也是在这个文件夹下进行的。
输入npm install。
运行hexo s -p 端口(端口自行修改，这里推荐4000)并在浏览器输入localhost:4000(其他端口也可以)，就能看到博客了！

1.3 利用Github Page为博客创建网址

此时的博客还只有你自己能看到，要想让别人看到，需要上传到Github Page上。
但在此之前，还需要做一些基础配置。
打开你的编辑器(我用的是sublime)，把博客文件夹拖进去/打开(对我来说就是jjl)。
打开站点配置文件(博客文件夹下的_config.yml)备用。
再次来到Github官网，新建一个项目(New Repository)，在这里强调一点：
$\color{red}{\Huge{\text{名字必须是”你的Github用户名”.github.io！名字必须是”你的Github用户名”.github.io！名字必须是”你的Github用户名”.github.io！}}}$
引号省略。

此时就跳转到了项目页面，点击旁边绿色的按钮(Clone or download)，复制它给的链接。

回到编辑器，拖到底部，把type后面的东西改成git,找到deploy:，加入repo:粘贴刚才复制的链接。
$\color{red}{\Huge{\text{.yml文件冒号后面一定要加空格！.yml文件冒号后面一定要加空格！.yml文件冒号后面一定要加空格！}}}$

现在，我们来上传并配置到博客。
输入hexo g和hexo d，其中g(generate)代表生成本地文件(预览)，d(deploy)代表上传。

然后…就大功告成啦！
